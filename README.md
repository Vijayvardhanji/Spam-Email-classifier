## 🎯 Project Overview

This project builds an end-to-end spam detection system:

1. **Loads** the SMS Spam Collection dataset (5,572 messages)
2. **Preprocesses** text using a full NLP pipeline
3. **Extracts features** with TF-IDF vectorization
4. **Trains** and **evaluates** two classifiers (Naive Bayes & SVM)
5. **Visualises** results with confusion matrices, accuracy comparisons, and word-frequency charts
6. **Serves** predictions through a beautiful Streamlit web app

## 🔬 NLP Pipeline

Each raw message goes through this preprocessing pipeline:

```
Raw Text
   │
   ▼
1. Handle Missing Values  → replace NaN with ""
   │
   ▼
2. Lowercase              → "FREE PRIZE!" → "free prize!"
   │
   ▼
3. Remove URLs            → strip http/https/www links
   │
   ▼
4. Remove Punctuation     → keep only [a-z0-9 ]
   │
   ▼
5. Tokenization           → word_tokenize() via NLTK
   │
   ▼
6. Stopword Removal       → remove "the", "is", "a", etc.
   │
   ▼
7. Porter Stemming        → "winning" → "win", "prizes" → "prize"
   │
   ▼
Clean Token String
```

## ⚙️ Feature Engineering – TF-IDF

**Why TF-IDF over Bag-of-Words?**

| Advantage | Explanation |
|-----------|-------------|
| **Penalises common words** | "the", "is" get low IDF → near-zero weight |
| **Rewards discriminative words** | "FREE", "winner", "prize" are rare overall → high TF-IDF |
| **Sparse & efficient** | Works natively with Naive Bayes & SVM |
| **Bigrams included** | `ngram_range=(1,2)` captures "free prize", "call now" |

**Parameters used:**

```python
TfidfVectorizer(
    max_features=5000,
    ngram_range=(1, 2),    # unigrams + bigrams
    sublinear_tf=True,     # log(1 + tf) dampening
    min_df=2               # ignore extremely rare terms
)
```

## 🏆 Models & Performance

### Naive Bayes (MultinomialNB)
- **Why:** Assumes feature independence (works well for text)
- **Pros:** Extremely fast, interpretable, great baseline
- **Alpha:** 0.1 (Laplace smoothing)

### SVM – LinearSVC + Calibration
- **Why:** Finds the maximum-margin hyperplane in TF-IDF space
- **Pros:** Higher precision, robust on imbalanced data
- **C:** 1.0 | Wrapped in `CalibratedClassifierCV` for probability outputs

### Results (Test Set – 20% split)

| Metric    | Naive Bayes | SVM       |
|-----------|-------------|-----------|
| Accuracy  | ~97.5%      | **~98.3%** |
| Precision | ~96.8%      | **~98.7%** |
| Recall    | ~93.4%      | **~95.2%** |
| F1-Score  | ~95.1%      | **~96.9%** |

> Exact numbers will vary slightly per run due to train/test splits.

## 💻 Usage

### Train Models

```bash
python src/train.py
```

This will:
- Preprocess the dataset
- Train both classifiers
- Print performance metrics
- Save models to `/models/`
- Save visualisations to `/screenshots/`

### Predict from CLI

```bash
# Single model
python src/predict.py "Congratulations! You've won a FREE prize. Call now!"

# Compare both models
python src/predict.py "Hey, are you free for lunch?" --both

# Specify model
python src/predict.py "Your account needs verification" --model naive_bayes
```

### Use in Your Own Code

```python
from src.predict import SpamPredictor

predictor = SpamPredictor(model_name='svm')
result = predictor.predict("FREE entry! Win £1000 cash prize now!")

print(result['label'])       # 'Spam'
print(result['confidence'])  # 0.9974
print(result['spam_prob'])   # 0.9974
print(result['ham_prob'])    # 0.0026
```
